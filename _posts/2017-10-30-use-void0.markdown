`a`태그에 클릭 이벤트를 바인딩할때 `href`속성값은 어떻게 설정하시나요?

## pattern 1 - `#`

```html
...
<a href="#" onClick="return handleClick()">Click me!</a>
...
<script>
function handleClick() {
  // statements
  return false;
}
</script>
...
```

## pattern 2 - `javascript:void(0);`

```html
...
<a href="javascript:void(0);" onClick="return handleClick()">Click me!</a>
...
<script>
function handleClick() {
  // statements
  return false;
}
</script>
...
```

둘 중에 어느 패턴이 조금 더 유용할까요?

`#`을 사용하는 경우 click 메서드 내부에서 href를 참조하는 기본 동작을 제어하지 않으면, location의 hash가 변경되고 맙니다. (또한 클릭 이벤트를 동적으로 binding 하는 경우도 마찬가지)

클릭 이벤트핸들러 실행시 `true` 를 반환해야 하는 경우 꽤 난감한 상황에 맞닥뜨리게 됩니다.

이렇게 여러가지를 고려하여 굳이굳이 `#`을 사용하기 보다는, `void(0)` 패턴을 사용하는 것이 더 낫지 않을까 하는 의견입니다. :-)
